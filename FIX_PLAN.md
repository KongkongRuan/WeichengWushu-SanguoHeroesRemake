## 问题1：菜单 ◀▶ 箭头一直左右抖动

### 根因
原版 `d(int,int,int)`（a.java:18758）确实让箭头位移 2px 闪烁，但原版主循环是 **10FPS**（a1019 每 100ms +1），`(a1019&1)<<1` 每 200ms 切换一次（2.5Hz 视觉）。H5 的 `frameCount` 是 60fps 计数（Game.ts:1872 `const blink = (this.frameCount & 1) << 1;`），每 33ms 切换一次 → 看起来就是持续抖动。

### 修复（1 行逻辑，3 处）
把所有箭头闪烁计数从 60fps 降为与原版一致的节拍：
- `game/src/core/Game.ts:1872`、`:1978`、`:2150`（战役点闪烁同理 :2134 附近）
```ts
// 改前: (this.frameCount & 1) << 1
// 改后: (Math.floor(this.frameCount / 12) & 1) << 1   // 60fps/12 = 5Hz, 与原版 100ms 逻辑帧一致
```
建议抽一个 `private get blink2px(): number { return (Math.floor(this.frameCount / 12) & 1) << 1; }` 统一替换。

### 原版渲染算法（a.java:18830-18995 的分派方法 + 11170-11200 的多点绘制 + 16946-16977 的帧推进）

**分派**（按塔类型 switch，c1107 塔槽：[2]=类型 [3]=等级 [4]=状态 [5]=朝向；var1/var2=世界像素 x,y；var4=当前帧）：

- **类型 0（石灰瓶）**：`img = t0[朝向+1]`；
  `drawX = x + y1125[朝向*4]`，`drawY = y + y1125[朝向*4+1] + 13`；
  `srcX = (朝向==0 ? 0 : y1125[朝向*4+2] * w1123[0][帧])`，`w/h = y1125[朝向*4+2/+3]`；
  若 3 ≤ 帧 < 10：再叠画 `t0_5` 于 `(drawX+2, y + y1125[朝向*4+1] - t0_5高 - w1123[0][帧] + 13)`（液位/喷溅条）。
- **类型 1, 6, 7**：switch 的 default 分支——静态绘制（a.java:18830-18890 前半部分本次未逐行读完，实现时先读该段；预计是 `t{type}_0` 整图直绘）。
- **类型 2, 3, 9**（单部件动画）：`img = t{type}[1]`；循环 i1122[type] 每个绘制点：
  `srcX = w1123[type][帧] * 帧宽`，绘制 `(帧宽,帧高)` 于 `(x+dx, y+dy+13)`（方法 a.java:11170）。
- **类型 5, 8**（双部件）：先同上画 `t{type}[1]`（用 type 的序列）；再画 `t{type}[2]`，**用 w1123[type-1] 的序列与帧宽高**；类型 5 额外 `drawY += x1124[帧]`。
- **类型 4（烟火）/10（装置）**：switch 前段（a.java:18830 附近）有独立 case，实现时先读原文确认（类型 4 与朝向相关：帧推进见下）。

**帧推进**（`c(int,int,int)` a.java:16946）：每逻辑帧 `帧+1`；`帧+1 > w1123[type].length - 3`（末 2 字节是宽高，故 -3）时归 0；**类型 0/4 归零时朝向随机化 `c1107[i][5] = rand&3`**。
