## 详细问题与修正

### 模块 01: `01_fields.java` — 字段声明问题

#### 问题 1.1: 缺失字段 `q1082` (boolean) ⚠️
- **问题**: VF L334 `private boolean q1082;` 在模块中缺失
- **影响**: `b(int,int,int)` 方法 (VF L15860) 使用 `this.q1082`, 缺失会导致编译错误
- **修正**: ✅ 已添加 `private boolean q1082;`

#### 问题 1.2: v2 注释错误归因 ⚠️
- **问题**: v2 修正注释错误地将字段遗漏归因于 "VF 字段声明遗漏"
- **实际情况**: 所有字段 (`a1038`→VF L101, `a1039`→VF L102, `e1025`→VF L260, `f1026`→VF L268, `at`→VF L162) 在 VF 中均存在, 是 v1 模块化时的遗漏
- **A1151 注释错误**: v2 说 "VF 误标为 int[]", 实际上 VF L30 正确为 `byte[]`, 错误在模块本身
- **修正**: ✅ 已修正所有注释, 正确标注 VF 行号

---

### 模块 03: `03_callbacks.java` — 回调方法问题

#### 问题 3.1: `run()` try-catch 范围过窄 ⚠️ 严重
- **问题**: try-catch 只包裹 `Thread.sleep(10)`, 但 VF 字节码异常表显示应覆盖整个循环体
- **VF 字节码异常表** (L27488-27502):
  ```
  try (41 -> 42): Exception   // System.currentTimeMillis
  try (43 -> 45): Exception   // G() 调用
  try (48 -> 50): Exception   // h1047=0
  try (50 -> 52): Exception   // repaint()
  try (52 -> 54): Exception   // serviceRepaints()
  try (57 -> 59): Exception   // sleep 参数
  try (59 -> 60): Exception   // Thread.sleep
  try (60 -> 61): Exception   // Thread.yield
  try (73 -> 74): Exception   // 100ms等待循环
  try (79 -> 81): Exception   // Thread.yield
  try (90 -> 91): Exception   // c1020 检查
  try (102 -> 104): Exception // s 计算
  try (104 -> 106): Exception // s 赋值
  try (114 -> 116): Exception // a1019 递增
  ```
- **异常处理器** (offset 84): `printStackTrace()` + `goto 36` (继续循环)
- **影响**: 原代码在 G() 或 repaint() 抛出异常时会崩溃, 修正后会打印堆栈并继续运行
- **修正**: ✅ 已将 try-catch 范围扩展到整个循环体

#### 问题 3.2: `keyPressed()` 注释错误 ⚠️
- **问题**: 注释说 "at 字段在 VF 字段声明中遗漏", 实际上 VF L162 有 `private int at;`
- **修正**: ✅ 已修正注释为 "at 字段在 VF L162 中存在"
