- **原因**: Vineflower 字段声明 L30 正确为 `byte[]`, 但模块化时误标为 `int[]`

### 模块 03: `03_callbacks.java` — 回调方法问题

#### 问题 3.1: keyPressed() 漏掉 `at` 检查 ⚠️ 严重
- **问题**: 缺少 `if (this.at != 0) return;` 防重复触发检查
- **VF 位置**: L27172 `var10000 = this.at; if (var2 == 0) { ... }`
- **影响**: 会导致按键重复触发, 破坏游戏状态机
- **字段确认**: `at` 字段在 javap L274 确认存在 (`private int at;`)
- **修正**: ✅ 已添加 `at != 0` 检查

#### 问题 3.2: hideNotify() 严重简化错误 ❌ 严重
- **问题**: 原代码只有 `this.b1174 = 0;` 一行
- **VF 位置**: L27115-27166 有复杂逻辑
- **正确逻辑**:
  1. 若 `cb==1500`: 设置 `C1175=true`
  2. 检查 `l` 状态, 仅在 46/48/47/2/12/13/22/23 状态处理
  3. 调用 `a(3)` 菜单跳转
  4. 若 `l1031`: 调用 `h(aq)` 恢复音频
  5. 重置 `r=0, at=0`
- **影响**: 游戏暂停/恢复逻辑完全失效
- **修正**: ✅ 已按 VF 完整恢复

#### 问题 3.3: run() 中 try-catch 缺少 printStackTrace
- **问题**: `catch (Exception e) {}` 空处理
- **VF 字节码**: L27443 确认有 `invokevirtual printStackTrace`
- **修正**: ✅ 已添加 `e.printStackTrace()`

## Vineflower 已知局限 (本次验证发现)

   - 这些字段在方法体中被使用, 但未在类顶部声明
   - 通过 javap 字节码交叉验证确认存在

2. **方法反编译失败**: `run()`, `b()`, `d()`, `av()`, `aw()`, `ay()` 等 6 个方法 VF 反编译失败
   - 已通过 javap 字节码手动恢复 (见模块 08)

3. **方法名/语义误判**: VF 输出的方法名与实际功能不符 (已在 v1 修正中处理)
