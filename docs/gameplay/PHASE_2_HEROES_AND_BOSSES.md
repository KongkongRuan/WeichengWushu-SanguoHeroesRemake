# 第二阶段方案：武将、名将与阵营特色

> 文档状态：概要完成，待第一阶段实玩后细化
> 依赖：[玩法增强总纲](../../GAMEPLAY_ENHANCEMENT_ROADMAP.md)
> 前置方案：[第一阶段：战术基础与局内决策](PHASE_1_TACTICAL_FOUNDATION.md)
> 预计开发量：约 2,500～4,500 行新增或重构代码
> 预计人工量：12～22 人日
> Token 区间：45 万～75 万
> 建议阶段预算：60 万 Token

## 1. 阶段目标

第二阶段让现有的“武将觉醒、敌方名将、国家和武文路线”从数值差异升级为可操作、可观察、可应对的战术系统。

完成后应交付：

- 一套可扩展的主动将技框架；
- 6 名己方武将的主动技能；
- 6 名敌方名将的独立特性；
- 蜀、魏、吴各两项候选阵营政令；
- 完整的预告、目标选择、冷却、存档和触屏操作；
- 将技、Boss 特性、军议和机关联动之间的组合测试。

## 2. 开始条件

第二阶段不得仅因第一阶段“代码写完”就开始，必须满足：

- classic/enhanced 规则隔离稳定；
- SeededRng、CombatEvents、ModifierResolver 和运行存档已完成；
- 第一阶段至少三张地图通过经典/强化实玩；
- 军议和增强联动的数值没有明显失控；
- 塔和敌人已经拥有稳定运行 ID；
- 战斗统计能正确归属直接、持续和联动伤害。

如果上述接口发生重大变化，先更新本方案的技术设计和预算。

## 3. 本阶段非目标

- 不把武将做成自由移动单位。
- 不一次性为全部 33 名己方武将制作主动技能。
- 不为所有敌将制作独立机制。
- 不增加装备、抽卡、养成星级或永久数值突破。
- 不实现历史地图事件和无尽模式。
- 不让主动将技消耗军令；军令仍是辅助系统，不是常规战斗蓝条。

## 4. 总体技术设计

### 4.1 推荐模块

~~~text
game/src/enhancement/skills/
├── ActiveSkillSystem.ts
├── SkillTargeting.ts
├── SkillEffects.ts
└── SkillRunState.ts

game/src/enhancement/bosses/
├── BossTraitSystem.ts
├── BossActions.ts
└── BossTraitState.ts

game/src/enhancement/factions/
├── FactionDecreeSystem.ts
└── FactionModifierHooks.ts

game/src/data/enhancement/
├── heroSkillData.ts
├── bossTraitData.ts
└── factionDecreeData.ts
~~~

### 4.2 将技模型

技能定义和技能运行状态分离：

~~~ts
type SkillTargetType = 'global' | 'point' | 'line' | 'area' | 'tower';

interface HeroSkillDefinition {
  id: string;
  heroId: number;
  targetType: SkillTargetType;
  cooldownTicks: number;
  chargesPerWave: number;
  range?: number;
  telegraphTicks: number;
  effectIds: string[];
}
~~~

技能冷却使用 10Hz 战斗逻辑帧：暂停时不推进，2 倍或 3 倍游戏速度下随战斗逻辑加速。UI 显示秒数时按逻辑帧换算。

### 4.3 Boss 特性与原版 bossType 分离

当前 bossType 已承担原版名将类型和免疫规则。新增 BossTraitId 必须单独存在，以 heroId/bossId 映射，不得重载 bossType 数字含义。

Boss 特性可以读取原版免疫，但不能覆盖或删除原版免疫。经典规则完全不创建 BossTraitState。

### 4.4 控制与反制原则

- 所有敌方主动行为至少提前 10 个逻辑帧预告。
- 玩家应能通过集火、保留将技、改变出兵时机或机关组合进行应对。
- Boss 不能无提示永久摧毁满级塔。
- 暂停塔、召唤、光环等效果必须有作用范围和剩余时间提示。
- 控制免疫不应覆盖整场；采用阶段、蓄力或血量阈值窗口。

## 6. 里程碑 P2.1：主动将技框架与目标选择

> 状态：待开发
> 建议预算：10 万 Token
> 合理区间：8 万～12 万 Token

### 6.1 解锁与入口

- 只有七级觉醒塔提供对应武将主动技能。
- 技能绑定塔 instanceId 和 heroId；塔被拆除后技能失效。
- 复用 BuildBar 中七级塔的保留动作槽展示“将技”。
- 移动端增加上下文将技按钮，只在选中可用觉醒塔时显示。
- 暂停菜单增加将技说明列表，但暂停状态不能释放技能。

### 6.2 目标选择

支持五种标准目标类型：

- global：确认后全局生效；
- point：选择一个地图点；
- line：从技能塔或指定起点选择方向；
- area：显示圆形或格状范围；
- tower：选择合法己方建筑。

目标选择复用现有 screenToLogical 和触屏拖动基础，但必须与建造模式互斥。进入将技目标模式后：

- 合法区域使用绿色或技能主题色；
- 非法区域显示红色；
- 返回键、右键和取消按钮均能无消耗退出；
- 第一次点击预览，第二次确认，避免触屏误放；
- 相机或画布缩放不改变逻辑目标。

### 6.3 冷却和次数

- 第一版默认每波最多 1 次，具体技能可配置。
- 冷却和波次次数同时满足才可释放。
- 军议可以修改冷却，但第一版不允许把冷却降到基础值的 50% 以下。
- 读档恢复剩余冷却、剩余次数和未完成预告；目标选择 UI 本身不保存，加载后回到安全待命状态。
- 技能释放事件写入战报，用于阶段验收。

### 6.4 验收标准

- [ ] 五种目标模板都有无具体英雄依赖的测试技能。
- [ ] 技能取消不消耗次数或冷却。
- [ ] 觉醒塔拆除、读档、关卡结束后状态正确释放。
- [ ] 1～3 倍速下冷却与战斗逻辑一致。
- [ ] 经典规则没有将技入口。

## 7. 里程碑 P2.2：首发 6 名己方武将

> 状态：候选内容，待 P2.0 冻结
> 建议预算：13 万 Token
> 合理区间：9 万～15 万 Token

### 7.1 候选阵容

为了覆盖三国、武文路线和不同目标模板，首发建议如下：

| 武将 | 技能草案 | 类型 | 战术用途 |
|---|---|---|---|
| 刘备 | 仁德 | global | 每场一次恢复 1 城防，并短暂鼓舞觉醒塔 |
| 关羽 | 青龙斩 | line | 对直线目标造成无视防御伤害，适合处理高甲纵队 |
| 诸葛亮 | 东风 | global/area | 延长火焰并引燃范围内油迹，强化火攻窗口 |
| 曹操 | 号令 | area | 一段时间提高范围内建筑攻击频率 |
| 郭嘉 | 奇佐 | area | 冻结普通敌人；Boss 只受到短减速或蓄力延迟 |
| 周瑜 | 火攻 | point/area | 点燃目标区域和滚油，对聚群敌人制造持续压力 |

首发名单不是永久锁定。P2.0 可根据觉醒顺序、阵营覆盖和第一阶段联动强度替换，但每个阵营至少有一名武系或君主、一名文系候选进入后续内容池。

### 7.2 平衡边界

- 单次将技不能直接清空同等级完整名将波。
- 恢复城防类技能每场最多 1 次。
- 全局增益持续时间原则上不超过 80 个逻辑帧。
- 伤害技能需要使用战斗统计的来源归属。
- 将技触发的火、冰、油联动按第一阶段 ReactionSystem 结算，不复制另一套逻辑。
- 技能不得绕过“使用军令后挑战成绩失效”的规则；将技是正常玩法，不算军令。

### 7.3 反馈

- 复用现有头像、名字和 Canvas 特效，不以新商业素材为前置条件。
- 释放前显示武将名与技能名，持续不超过操作所需。
- 范围、持续时间和伤害归属在战报可见。
- 若资源允许，可为每个技能增加独立颜色和短音效；缺失时使用统一可读反馈。

### 7.4 验收标准

- [ ] 6 个技能均可从实际觉醒塔释放。
- [ ] 每个技能至少有正常、无目标、Boss 和读档测试。
- [ ] 技能与 12 项军议、4 项增强联动不存在明显无限循环。
- [ ] 三国各完成至少一场主要依赖将技时机的实玩。

## 8. 里程碑 P2.3：首发 6 名敌方名将特性

> 状态：候选内容，待 P2.0 冻结
> 建议预算：15 万 Token
> 合理区间：10 万～18 万 Token

### 8.1 候选名将

根据现有 BOSS_LIST_Z1131，首发优先覆盖前期群雄和中期跨阵营敌将：

| 名将 | 特性草案 | 预告与应对 |
|---|---|---|
| 张角 | 撒豆成兵 | 定期召唤无金币幻兵；预告后用范围伤害识别和清理 |
| 华雄 | 破阵冲锋 | 血量阈值触发短冲锋；预留控制或爆发打断蓄力 |
| 貂蝉 | 离间 | 暂停附近一座建筑；提前标记目标，可用将技或拆建调整 |
| 吕布 | 无双 | 周期性短暂无视控制；避开窗口，窗口结束后集中控制 |
| 董卓 | 暴政 | 给附近普通敌军护甲光环；优先集火或用无视防御攻击 |
| 周瑜 | 火势 | 周期性强化燃烧敌军并削弱冻结；利用沸水灭火和错峰控制 |

### 8.2 通用 Boss 行为状态

BossTraitState 至少包括：

- traitId；
- phase；
- cooldownTicks；
- telegraphTicks；
- activationCount；
- auraTargetIds 或召唤来源；
- 已触发血量阈值；
- 种子随机状态引用。

不得把这些状态塞进 Enemy.effect 或复用原版动画字段。

### 8.3 召唤和控制规则

- 召唤单位计入敌人池上限，池满时延迟召唤，不覆盖现有敌人。
- 幻兵不掉金币、不计正常击杀奖励，但计战报中的召唤清理数。
- 暂停塔不能阻止玩家出售或查看；是否允许升级由规格冻结。
- Boss 光环只影响有效范围内当前敌军，离开范围立即失效。
- 血量阈值行为每个阈值只触发一次，读档后不得重复。

### 8.4 验收标准

- [ ] 6 个特性均有 1 秒左右的可见预告。
- [ ] 每个特性至少有一种可靠玩家应对方式。
- [ ] 原版 bossType 免疫测试全部保持通过。
- [ ] 敌人池满、Boss 死亡、读档和清场军令时状态正确。
- [ ] 同一 Boss 不因 3 倍速跳过预告或重复触发。

## 9. 里程碑 P2.4：阵营政令

> 状态：设计草案
> 建议预算：6 万 Token
> 合理区间：4 万～7 万 Token

### 9.1 选择流程

- 仅强化规则出现。
- 在国家和武/文路线选择完成后、进入战役地图前选择一项政令。
- 每个国家提供两项，当前王者之路流程内保持不变，直至重新开始该路线。
- 自由模式每次开战前可重新选择。
- 战役地图和暂停菜单显示当前政令。

### 9.2 第一版候选

| 阵营 | 政令 | 效果方向 |
|---|---|---|
| 蜀 | 同袍协力 | 将技冷却小幅降低，觉醒塔互相提供有限支援 |
| 蜀 | 火计连环 | 火焰和冰火爆裂加强，但纯物理伤害不加成 |
| 魏 | 法度森严 | 麻痹、减速等控制更稳定，但爆发伤害不加成 |
| 魏 | 工造有序 | 前若干次升级便宜，拆除回收略降低 |
| 吴 | 强弓水军 | 远程和液体机关范围小幅提高 |
| 吴 | 水火并济 | 油火、沸水和寒冰联动获得额外收益 |

具体数值必须等第一阶段战报和 P2.2 将技强度确定后再冻结。

### 9.3 验收标准

- [ ] 六项政令都通过 ModifierResolver 生效。
- [ ] 经典规则与原版武/文路线不受影响。
- [ ] 政令保存、继续游戏和自由模式重选行为正确。
- [ ] 三国至少各存在两种可辨认的强化构筑方向。

## 10. 里程碑 P2.5：组合回归、无障碍与阶段验收

> 状态：待开发
> 建议预算：10 万 Token
> 合理区间：7 万～12 万 Token

### 10.1 组合矩阵

至少覆盖：

- 6 将技 × 普通兵/Boss；
- 6 Boss 特性 × 清场军令/读档/3 倍速；
- 6 政令 × 12 军议；
- 将技 × 4 增强联动；
- Boss 免疫 × 将技控制；
- 塔拆除/重新觉醒 × 技能状态；
- 强化规则和经典规则完整战役流程。

不要求穷举所有组合，但高风险交互必须形成表驱动测试。

### 10.2 可读性

- 将技就绪、不可用、目标非法和冷却必须有不同形态。
- Boss 预告不能只依靠颜色；同时提供图标或文字。
- 小屏上 Boss 横幅不能遮挡将技确认。
- 多个效果同时出现时，以 Boss 预告和玩家目标选择为最高显示优先级。

### 10.3 阶段完成标准

- [ ] P2.0～P2.4 全部验收完成。
- [ ] 6 名己方武将和 6 名敌方名将进入正式内容池。
- [ ] 3 个阵营各有可辨认的强化路线。
- [ ] npm test 和 npm run build 通过。
- [ ] 三国各至少完成两张强化战役实玩。
- [ ] 没有无提示秒杀、永久控塔或不可恢复软锁。
- [ ] 经典模式没有将技、Boss 特性和政令入口。
- [ ] 将实玩数据、调整和实际 Token 消耗回写本文。
- [ ] 冻结第三阶段可依赖的技能、Boss 和修正器接口。

## 11. 风险与回退

| 风险 | 控制措施 | 回退方案 |
|---|---|---|
| 触屏目标选择困难 | 二次确认、吸附合法格、统一目标模板 | 第一版只开放 global/area 技能 |
| 技能与军议叠加失控 | 冷却下限、次数上限、统一修正器 | 单项组合互斥配置 |
| Boss 令人挫败 | 强制预告、持续上限、明确反制 | 降级为被动光环 |
| 召唤突破 80 敌人池 | 延迟生成、严格来源和奖励规则 | 张角改为已有敌人分身视觉 |
| 原版免疫被破坏 | BossTrait 与 bossType 分离 | 关闭对应增强控制结果 |
| UI 信息过载 | 优先级队列、摘要和详情分层 | 暂停菜单集中显示说明 |
